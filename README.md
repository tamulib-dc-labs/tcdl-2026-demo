# Whisper Transcript Reviewer

A web application for reviewing VTT files and Whisper / Whisper X JSON transcripts with word-level confidence scoring.

## Using This Template

This repository is a GitHub template. To create your own instance:

1. Click **"Use this template"** > **"Create a new repository"** on GitHub
2. Edit `config.yml` to customize your site (you can do this directly in GitHub's web interface):

```yaml
site:
  title: "My Transcript Reviewer"
  subtitle: "Your custom subtitle here"
  footer_text: "Your Footer Text"
  theme: "synthwave"  # See themes section below
```

3. Add your media files to `public/config.json` (see format below)

4. Enable GitHub Pages:
   - Go to **Settings** > **Pages**
   - Under **Source**, select **GitHub Actions**

5. Push your changes - the site will automatically build and deploy!

Your site will be available at: `https://YOUR-USERNAME.github.io/YOUR-REPO-NAME/`

## Configuration

### Site Configuration (config.yml)

Edit `config.yml` to customize your site's appearance:

| Setting | Description | Default |
|---------|-------------|---------|
| `site.title` | Page title and header text | "Whisper Transcript Reviewer" |
| `site.subtitle` | Subtitle below the header | "Review VTTs Generated by Whisper or Whisper X." |
| `site.footer_text` | Footer text | "Whisper JSON Viewer" |
| `site.theme` | DaisyUI theme name | "synthwave" |

Optional build settings:

```yaml
build:
  base_path: "/custom-path/"  # Override auto-detected path
```

The base path is automatically detected from your repository name, so you typically don't need to set this.

### Media Files (config.json)

Add your transcript files to `public/config.json`. Each entry should have:

```json
[
  {
    "audio": "https://example.com/media/audio.m3u8",
    "url": "https://example.com/transcripts/transcript.json",
    "vtt": "https://example.com/vtts/subtitle.vtt",
    "name": "display-name"
  }
]
```

| Field | Required | Description |
|-------|----------|-------------|
| `audio` | Yes | URL to audio/video stream (M3U8, MP4, etc.) |
| `url` | Yes | URL to Whisper X JSON transcript |
| `vtt` | No | URL to VTT subtitle file |
| `name` | Yes | Display name in the dropdown selector |

See `public/config.example.json` for a sample.

### Transcript JSON Format

The transcript JSON files can include optional word-level data with confidence scores:

```json
{
  "word_score_buckets": {
    "Good": 0.85,
    "Neutral": 0.65,
    "Bad": 0.25
  },
  "segments": [...]
}
```

This is useful to add to your pipeline when you have items with particularly bad confidence scores.

If `word_score_buckets` is not included, defaults will be used instead.

## Local Development

```bash
npm install         # Install dependencies
npm run dev         # Start development server
npm run build       # Build for production
npm run preview     # Preview production build locally
```

## Deployment

This project automatically builds and deploys via GitHub Actions when you push to the main branch.

**Important:** You must configure GitHub Pages to deploy from **GitHub Actions** (not from a branch):
1. Go to **Settings** > **Pages**
2. Under **Source**, select **GitHub Actions**

The build process:
1. Reads your `config.yml` settings
2. Updates the HTML with your title, subtitle, theme, and footer
3. Sets the correct base path from your repository name
4. Builds and deploys to GitHub Pages

## Themes

- synthwave, retro, cyberpunk, valentine, halloween, garden
- forest, aqua, lofi, pastel, fantasy, wireframe, black
- luxury, dracula, cmyk, autumn, business, acid, lemonade
- night, coffee, winter, dim, nord, sunset

Change the theme by editing `site.theme` in `config.yml`.
